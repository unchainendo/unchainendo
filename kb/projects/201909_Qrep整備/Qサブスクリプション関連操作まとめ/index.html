Qサブスクリプション関連操作まとめ
2020/03/01

Qサブスクリプションの作成

前提条件

  • ソース表のユニーク索引が定義された列に重複レコードが存在しないこと
  • ソース表とターゲット表の列数、列名に差異がないこと
  • ソース表に対する読み取り権限がフルリフレッシュ用ユーザーに付与されていること

Qサブスクリプション定義用スクリプトの作成

Qsub定義用のスクリプトを作成する。 複数のQsub定義を1ファイルにまとめても良い。 SET RUN SCRIPT NOW STOP ON SQL ERROR ONとしているのでasnclpコマンド実行と同時に制御表が更新される。 SUBS表、TARGETS表に入る設定はQsub定義時に指定する。 ここでは

cat > /home/db2qrep/QREP/define/create_qsub_<Qsub名>.txt << EOF
ASNCLP SESSION SET TO Q REPLICATION;
SET PWDFILE "<パスワードファイル(フルパス)>";

SET SERVER CAPTURE TO DB <ソースDB名>;
SET SERVER TARGET TO DB <ターゲットDB名>;

SET CAPTURE SCHEMA SOURCE <キャプチャー制御表スキーマ名>;
SET APPLY SCHEMA <アプライ制御表スキーマ名>;

#SET RUN SCRIPT LATER;
SET RUN SCRIPT NOW STOP ON SQL ERROR ON;

CREATE QSUB USING REPLQMAP "<Qマップ名>" (SUBNAME "<Qサブスクリプション名>" "<ソース表スキーマ名>"."<ソース表名>" OPTIONS
HAS LOAD PHASE I CAPTURE_LOAD R REPLICATE ADD COLUMN Y START AUTOMATICALLY NO EXIST TARGET NAME "<ターゲット表スキーマ名>"."<ターゲット表名>"
CONFLICT ACTION F ERROR ACTION S);
EOF

以下はまた時間があるときに。。

  • Qサブスクリプションの削除
  • Qサブスクリプションの起動・停止
