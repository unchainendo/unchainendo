FederationServer環境構築手順
2020/03/01

作業前提

環境区分

  • ホスト
    ソースDBが存在する環境
  • DBサーバー
    ターゲットDBが存在する環境

ソフトウェアバージョン

  • ホスト
    • Db2 V10 CM8
  • DBサーバー
    • Db2 V11.5

1. FederationServer設定

FederationServer用ユーザーの作成

FederationServer用インスタンスのオーナーをfedmin、NICKNAME定義用ユーザーをfedsysとする。 FederationServer用DBのパス(今回は/data/db2fed)にアクセス権があることを確認しておく。なければ付与する。 以降の作業は全てDBサーバー上で実施する。

FederationServer用インスタンスの作成

ターゲットDBと同一インスタンスに構成しても良いが、今回はターゲットDBに極力影響しない範囲で実装するためインスタンスごと分ける。

# インスタンスオーナーにスイッチ
su - fedmin

# FederationServer用インスタンスの作成
# /opt/IBM/db2/V11.5/instance/db2icrt -u <インスタンスオーナー> <インスタンス名>
/opt/IBM/db2/V11.5/instance/db2icrt -u fedmin fedmin 

# インスタンス確認
/opt/IBM/db2/V11.5/instance/db2ilist

Db2ライセンス適用

Developer-Cライセンスを削除してStandard Editionライセンスを適用する。
手順はQアプライ環境構築手順参照。

FederationServer用インスタンスの起動

db2start

ターゲットDB、ソースDBのカタログ

FederationServerとしてはソースDBしか見に行かないが、同じ環境からターゲットDBにも接続できた方が export→diffの操作が楽になるので両方ともカタログしておく。 手順はQアプライ環境構築手順参照。

DBM構成パラメータの変更

作成したインスタンスをFederationServerとして使うための設定。

# DBM構成パラメータの変更
db2 update dbm cfg using FEDERATED YES

# インスタンス再起動(設定反映のため)
db2stop
db2start

# DBM構成パラメータの確認
db2 get dbm cfg | grep FEDERATED

FederationServer用DBの作成

無駄な文字コード変換を避けるためターゲットDBと同じコードセットで作成する。(作成後は変更不可) ページサイズは32Kとする。order byしてexportする際に一時表スペースが溢れないようにするため。
以降、FederationServer用DBは「FEDDB」とする。

# FederationServer用DBの作成
# db2 create db <DB名> on <データパス> using codeset <コードセット> territory JP collate using identity pagesize 32K
db2 create db FEDDB on /data/db2fed using codeset IBM-943 territory JP collate using identity pagesize 32K

# DB一覧の確認
db2 list db directory

NICKNAME定義用ユーザーへの管理者権限付与

NICKNAME作成やFederationServer経由のexportはNICKNAME定義用ユーザーで行うため、 FEDDBの管理者権限をNICKNAME定義用ユーザーへ付与する。

# FEDDBへdb2connect
db2 connect to FEDDB

# NICKNAME定義用ユーザー(fedsys)にDBADM権限を付与
# db2 grant <権限> on database to user <ユーザー名>
db2 grant dbadm on database to user fedsys

# 権限が付与されたことを確認
# db2 "select GRANTEE, DBADMAUTH from syscat.dbauth where GRANTEE='<ユーザー名>'"
db2 "select GRANTEE, DBADMAUTH from syscat.dbauth where GRANTEE='FEDSYS'"

# NICKNAME定義用ユーザーにDb2用環境変数定義を追加
su - fedsys
# 以下の4行を追記
# # The following three lines have been added by UDB DB2.
# if [ -f /home/fedmin01/sqllib/db2profile ]; then
#     . /home/fedmin01/sqllib/db2profile
# fi
vi ~/.bashrc

ラッパー、サーバー、ユーザーマッピングの作成

ホストのソースDBに接続するためのwrapper, server, user mappingを作成する。 以降の作業はNICKNAME定義用ユーザー(fedsys)で行う。
serverはNICKNAME作成時に指定する。
これでFederationServerとしての環境設定は完了。

# wrapperの作成
# db2 create wrapper <wrapper名>
db2 create wrapper drda

# wrapperが作成されたことを確認
db2 "select * from syscat.wrappers"

# serverの作成
# db2 "create server <server名> type DB2/zOS version 10.1 wrapper <wrapper名> authorization <ソースDB接続ユーザー名> password <パスワード> options (node '<ノード名>',DBNAME '<DB名>')"
db2 "create server HOST_SOURCE type DB2/zOS version 10.1 wrapper drda authorization qrep_source password xxxxxx options (node 'HOST',DBNAME 'SOURCEDB')"

# serverが作成されたことを確認
db2 "select * from syscat.servers"

# user mappingの作成
# create user mapping for USER server <server名> options(remote_authid '<ソースDB接続ユーザー名>',remote_password '<パスワード>')
create user mapping for USER server HOST_SOURCE options(remote_authid 'qrep_source',remote_password 'xxxxxx')
